Multi-Tenant Login and Role-Based Access Plan

You can implement multi-tenant login and role-based access in DuraiPricingTool. Plan below is structured for your current FastAPI + React stack.

1. Target Model

  1. Tenant is isolated business context (data + users + policies).
  2. User can belong to one or more tenants.
  3. Roles are tenant-scoped.
  4. Every request resolves tenant_id and enforces permissions.

2. Data Model Plan

  1. Add tables: tenants, users, roles, permissions, user_tenant_roles, refresh_tokens.
  2. Add tenant_id to business tables (quotes, quote_line_items, workflow_rules, master/config tables).
  3. Add composite indexes with tenant_id.

3. Authentication Plan

  1. Use internal JWT login now; keep OIDC/SAML-ready extension path.
  2. Add endpoints: /auth/login, /auth/refresh, /auth/logout, /auth/me, /auth/tenants.
  3. Token claims include sub, tenant_id, role_ids, permissions, exp.

4. Authorization Plan

  1. Define roles: SuperAdmin, TenantAdmin, PricingManager, SalesRep, Approver, Analyst, Viewer.
  2. Define permissions like quotes.read, quotes.write, quotes.approve, admin.workflow.manage.
  3. Implement FastAPI guards: require_auth, require_tenant, require_permission.

5. API Enforcement Plan

  1. Resolve tenant context from token and reject missing/invalid tenant.
  2. Apply tenant filters to all queries.
  3. Block cross-tenant reads/writes.
  4. Audit privileged actions.

6. Frontend Plan

  1. Add login page and secure token storage flow.
  2. Add tenant switcher UI after login.
  3. Add route-level permission guards.
  4. Hide/disable unauthorized actions in Admin/Quote screens.

7. Migration Plan

  1. Create schema migrations for auth and tenant columns.
  2. Backfill existing rows to default tenant.
  3. Then enforce tenant_id NOT NULL and strict filters.

8. Testing Plan

  1. Unit tests for token and permission logic.
  2. API tests for tenant isolation and role matrix.
  3. E2E tests for login, tenant switch, role-based UI behavior.

9. Rollout Plan

  1. Week 1: schema + auth base.
  2. Week 2: backend permission/tenant enforcement.
  3. Week 3: frontend login + switcher + guards.
  4. Week 4: pilot onboarding + hardening + audit review.
