Pharma Client Implementation Report

Client profile: Global pharmaceutical manufacturer with multi-region production plants and regional customer quoting needs.
Platform: DuraiPricingTool (FastAPI + DuckDB/Postgres hybrid + configurable formula/workflow engine)

1) Implementation Start Plan (First 4 Weeks)

Week 1: Data Foundation and Regional Model

  1. Define master data for products, customers, sellers, and plant-region attributes.
  2. Include pharma fields: dosage form, strength, pack size, cold-chain flag, controlled-substance flag, plant code, source country, transfer price, lead time, GMP status.
  3. Load baseline data via POST /master/seed-sample-large and POST /admin/data/import/{table_id}.
  4. Enable hybrid mode for transactional + analytics scale (DB_ENGINE=hybrid).

Week 2: Quote Structure and Formula Layer

  1. Configure line-item columns using GET/PUT /admin/line-item-config.
  2. Add mandatory quote fields: region, country, incoterm, tender_id, contract_type, fx_rate_date.
  3. Add line fields: batch_risk, shelf_life_months, plant_source, cold_chain_cost, regulatory_fee.
  4. Implement formulas via POST /quotes/calculate and reusable logic via POST /admin/field-logic/save.

Week 3: Workflow and Approval Governance

  1. Configure approvals by region and channel using POST /quotes/workflow/evaluate and workflow_rules.
  2. Enforce hard stops for non-compliant deals (low margin, invalid shelf-life, unauthorized controlled-substance channel).

Week 4: Pilot and KPI Validation

  1. Pilot in two regions first (example: NA and EU), then expand.
  2. Track KPIs: quote cycle time, approval rate, margin leakage, win rate by region/channel.
  3. Validate with /quotes/list and analytics endpoints (/analytics/waterfall, /bar, /time-series, /drilldown).

2) Recommended Business Rules for Multi-Region Pharma Quoting

A. Price Construction Rules

  1. Base Price: Regional list price by SKU and country.
  2. Plant Sourcing: Select source plant by regulatory approval, capacity, and landed cost.
  3. Landed Cost: landed_cost = ex_plant_cost + freight + cold_chain_cost + duty + insurance.
  4. FX Rule: Convert using approved treasury rate; lock rate for quote validity.
  5. Net Price: net_price = list_price - contract_discount - volume_discount - tender_discount + surcharge.

B. Margin and Compliance Guardrails

  1. Define minimum gross margin thresholds by region and channel.
  2. Block quote if controlled-substance license or eligibility is missing.
  3. Require QA approval when cold-chain route/lead-time violates stability constraints.
  4. Enforce minimum remaining shelf-life percentage at delivery.

C. Volume, Tender, and Contract Rules

  1. Apply discounts only from approved tier tables.
  2. Allow tender contracts to override commercial list only within approved floor rules.
  3. Separate off-invoice rebates from invoice discounts for compliance and analytics transparency.

D. Intercompany and Multi-Plant Rules

  1. Set transfer price floor by source plant and legal entity.
  2. Route for finance approval when transfer floor would be breached.
  3. Trigger regional finance review when plant-country mismatch occurs.

E. Workflow Rules

  1. Auto-approve when margin and discount are within policy.
  2. Sales manager approval for moderate threshold breaches.
  3. Regional pricing head approval for large deals or tender exceptions.
  4. Finance/CFO approval for floor breaches, high FX volatility, or intercompany exceptions.
  5. QA/Regulatory co-approval for cold-chain risk and controlled-substance products.

3) DuraiPricingTool Configuration Blueprint

  1. Add master data tables: regional_price_list, plant_costs, fx_rates, tender_contracts, regulatory_rules, channel_discounts, freight_lanes.
  2. Apply formula order: cost build -> discounts/surcharges -> net price -> margin -> validations.
  3. Use validation levels: warning for policy deviation, error for compliance breach.
  4. Persist audit trail: rule version, approver role, timestamp, and reason.
  5. Add analytics: margin bridge by region, plant-source impact, tender vs non-tender profitability.

4) Target Outcome

A controlled multi-region quote process with region-aware pricing, plant-aware cost-to-serve logic, compliance-first approvals, and measurable margin protection across global pharma operations.
