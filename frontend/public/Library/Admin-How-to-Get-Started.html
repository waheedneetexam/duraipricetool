Admin How To Get Started

This reference captures the steps for a newly onboarded admin to bring up the multi-tenant pricing platform.

1. Initial Access and Verification

  1. Ensure you have a Tenant Admin or SuperAdmin role via the auth console.
  2. Visit /auth/login with your assigned email/password and confirm you can call /auth/me.
  3. Switch tenants (if multi) using the drop-down in the header; it triggers the X-Tenant-ID header on future calls.

2. Tenant Configuration Checklist

  1. Open Admin > Table Manager and load the defaults (line item, logic, AI templates).
  2. Seed master data for the new tenant when data is not yet available: /admin/seed/sample-data?row_count=1000.
  3. Run the workflow seeder /admin/seed/workflow-rules to populate tenant-specific approvals.
  4. If ingestion is required, upload first CSV chunk through the CSV Upload panel (tenant context is automatic).

3. Creating and Configuring a New Tenant

  1. Create the tenant record in the database (tenants table) and ensure tenant_id is unique.
  2. Assign the bootstrap SuperAdmin via user_tenant_roles referencing the new tenant.
  3. Create master data entries (products/customers/sellers) with tenant_id set to the new tenant when using /master APIs.
  4. Upload tenant-specific line item config and field logic (tabs in Admin screen) — this is now tenant bound.

4. Operational Steps for a New Tenant

  • Publish scope: ensure quotes and workflow_rules have tenant filters applied.
  • Set pricing rules/state machine: use /admin/line-item-config and /admin/field-logic.
  • Run analytics smoke via /analytics/waterfall (tenant_id sent automatically). If data is not yet seeded, use CSV or sample-data seeder.
  • Document tenant-specific KPIs; the platform’s embedded waterfall and KPI widgets provide summary data when seeded.

5. Troubleshooting

  • If a call returns 403 Missing permission, confirm your role includes the required permission flag (admin.manage or quotes.write).
  • Use /auth/refresh automatically handled by the UI; if you observe sticky 401s, clear local storage and log in again.
  • Double-check tenant header by viewing X-Tenant-ID in browser devtools when saving configs.
