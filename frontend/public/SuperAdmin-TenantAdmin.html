SuperAdmin vs TenantAdmin: What They Can Do

Context: multi-tenant DuraiPricingTool with tenant-scoped roles and permissions.

Role Comparison

Capability SuperAdmin TenantAdmin
Tenant Management Create, suspend, reactivate, and delete tenants. Manage only the assigned tenant settings.
User Management Create platform users; assign users across tenants. Create users inside own tenant only.
Role Assignment Grant/revoke any role, including TenantAdmin. Grant tenant-level business roles (not SuperAdmin).
Access Scope All tenants and global administration APIs. Single tenant data and configurations.
Master Data Can view/fix any tenant master data issues. Full CRUD on own tenant master data.
Workflow Rules Can enforce global guardrails/templates. Configure approval rules for own tenant.
Quotes Cross-tenant support view (audited), emergency override policy. Full quote operations within tenant policy limits.
Audit & Compliance Platform-wide audit access and compliance export. Tenant-level audit access only.
Security Policies Set platform defaults (password, SSO, token policy). Can apply tenant options within platform limits.
Billing / Licensing Manage subscription, usage limits, and plan controls. View tenant usage; no cross-tenant billing actions.

Recommended Permission Keys

SuperAdmin: platform.tenants.manage, platform.users.manage, platform.audit.read, platform.security.manage

TenantAdmin: tenant.users.manage, tenant.roles.assign, tenant.masterdata.manage, tenant.workflow.manage, tenant.audit.read

Hard Restrictions

  1. TenantAdmin must never access another tenant’s rows.
  2. TenantAdmin cannot assign or remove SuperAdmin role.
  3. Cross-tenant support by SuperAdmin must be fully audited.
